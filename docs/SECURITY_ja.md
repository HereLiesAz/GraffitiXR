# セキュリティポリシー

## サポートされているバージョン
現在、セキュリティアップデートは `main` ブランチに対してのみ提供されています。

## 脆弱性の報告
GraffitiXR内にセキュリティの脆弱性を発見した場合は、公に開示しないでください。
代わりに、GitHubでプライベートなセキュリティアドバイザリを開くか、プロジェクトのメンテナーに直接メールしてください。
脆弱性の報告には48時間以内に対応し、問題の理解と解決に向けてお客様と協力するよう努めます。

## 脅威モデルのコンテキスト
GraffitiXRは、ローカルデバイス上で完全に動作するオフラインファーストのアプリケーションとして設計されていることに注意してください。バックエンドサーバーへの接続、ユーザーデータの送信、または認証資格情報の処理は行いません。

セキュリティレポートは主に以下につながる可能性のある問題に焦点を当てる必要があります：
- ローカル権限昇格
- 悪意のあるプロジェクトファイル (.gxr) による悪用
- ネイティブC++レンダリングエンジンにおける脆弱性（バッファオーバーフローなど）
