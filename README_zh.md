# GraffitiXR

GraffitiXR 是一款为街头艺术家设计的 Android 应用程序。有很多应用程序可以在您的相机视图上叠加图像以便您进行虚拟描摹，但是当我在手机上保存的草图基础上绘制壁画时，使用三脚架真的很破坏节奏。我们到处走动。我呢，我把手机放在口袋里。即使是使用 AR 保持图像稳定在原位的应用程序也无法应对口袋里无尽的黑暗。

因此，我通过重新利用（内行人称之为）网格方法来制作更好的东西。我一直在想：“为什么不能像持久锚点一样保存这些特定的涂鸦，以便覆盖层始终准确地位于正确的位置？”

所以，现在，这就是那些涂鸦的作用。

我不得不发明一个无需借助云端即可在 Android 上运行的自定义高斯泼溅引擎——因为涂鸦，你知道的，是非法的。

我接着使用了我称之为“目的论 SLAM”的技术——因为我们知道结果应该是什么样子，所以我使用 OpenCV 来查看您的进度，这意味着您进行的越深入，覆盖层与墙壁的贴合度就越高。如果没有这个，您将用画作本身掩盖那些标记，使应用程序在您进行过程中变得越来越不准确。这正是其他类似应用程序真正失败的地方。

只是为了好玩，我包含了非 AR 图像叠加功能来进行图像描摹，就像您使用那些其他应用程序一样，以防您喜欢那种方式。或者，如果您非常疯狂，还有效果图模式。拍一张墙壁的照片，然后我提供了一些用于快速制作效果图的快速工具。并且如果您无须证明什么，您只是想完美地将某些内容复制到纸上，透写台模式允许您将手机用作透写台，保持屏幕常亮并调高亮度，将您的图像锁定到位并阻止所有触摸，直到您完成。

然后，还有一套不错的设计工具，支持多层图形创建。我可以继续说下去，但我觉得我已经说得够多了。

**GraffitiXR** 是一款为街头艺术家设计的离线 Android 应用程序。它使用 AR 通过基于置信度的体素映射系统将图像投影到墙壁上。

## 主要特点
*   **离线优先：** 无云依赖；零数据收集。
*   **自定义引擎 (MobileGS)：** 用于 3D 高斯泼溅和空间映射的 C++17 原生引擎。
*   **完整的 ARCore 管道：** 通过 `BackgroundRenderer` 提供实时摄像头画面，颜色帧重新定位，以及 ARCore Depth API — 所有这些都向 SLAM 引擎提供真实数据。
*   **AzNavRail UI：** 拇指驱动的导航，适合现场单手使用。
*   **单一 GL 渲染路径：** `ArRenderer` 在单个 `GLSurfaceView` 中处理摄像头背景 (`BackgroundRenderer`) 和 SLAM 体素泼溅 (`slamManager.draw()`)。
*   **多镜头支持：** 在受支持的设备上自动使用双摄像头立体深度；回退到光流。
*   **目的论校正：** 使用 OpenCV 指纹识别进行自动地图到世界对齐。

## 架构
严格解耦的多模块架构：
*   `:app` — 导航，`ArViewport` 组合，相机所有权编排。
*   `:feature:ar` — ARCore 会话，`ArRenderer`，`BackgroundRenderer`，传感器融合，SLAM 数据输入。
*   `:feature:editor` — 图像处理，图层管理。
*   `:feature:dashboard` — 项目库，设置。
*   `:core:nativebridge` — `SlamManager` JNI 桥接，`MobileGS` 体素引擎，OpenGL ES 渲染。
*   `:core:data` / `:core:domain` / `:core:common` — 清洁架构数据层。
